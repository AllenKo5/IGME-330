Noteworthy

  • Mobile Friendly: App functions on mobile and tablet, though most testing was done on desktop.
  • Toggle Favorites: You can both favorite and unfavorite cards in the app without having to switch pages.
  • Multiple Pages: If search results are greater than your max results, they are split into multiple pages.
  • Multiple Favorites/Community Pages: If there are more than 10 items stored in the favorites/community page, they are split into multiple pages.
  • Back to Top: Implemented a back to top button to quickly get back to the top of the screen.

Grading

  • Local Storage: Saves last term searched by user, as well as max results. Additionally, remembers where the user last left off while searching.
  • Web Components: Has 6 web components - header, footer, navbar, and 3 different types of cards.
  • Fetch: Uses Ajax via fetch() in the app.
  • Firebase: Uses Firebase to keep track of total likes in the community page.
  • Labels: Easy to use with instructions, buttons and inputs are all labelled.
  • Activity Indicators: Search button has a spinner when searching, and favorite buttone changes to reflect state.
  • Fails Gracefully: Displays "no results found" when there aren't any valid results.
  • Has Impact: Serves as a way to search for Yu-Gi-Oh! cards on browser, something that is surprisingly difficult.

To Do

  • Updated cards: Cards would just be images, and clicking on them would have "pop-ups" to display more information.
  • Sorting: The ability to sort the cards in the app and favorites, as well as sorting the community tab by likes.
  • Deck Building: Users can create decks with the card search, though I might have run into image rate limits.
  • General Polish: Some of the code was a little sloppy and inconsistent across js files.

Proposal

API

The API used for this project is the YGOPRODeck API.

Mockup

Functionality

Users will be able to search for specific cards using a series of parameters, including card name, card supertype/subtype, Type, Attribute, Level/Rank, ATK/DEF, etc. These cards can then be selected to display information about them, as well as being able to favorite them and save them to a community page displaying the most favorited cards.
